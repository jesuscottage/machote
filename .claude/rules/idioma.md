# Rule: Documentation Language / Regla: Idioma de Documentación

**Priority / Prioridad**: High — applies to all sessions and agents.

## Configuration / Configuración

```yaml
# Change this to match your project's primary language
# Cambia esto según el idioma principal de tu proyecto
idioma-principal: es
```

## Main rule / Regla principal

**When `idioma-principal: es`**: All documentation written in this project must be in Spanish.
**When `idioma-principal: en`**: All documentation written in this project must be in English.

This includes without exception:
- Comments in source code files
- Analysis files (`_ANALYSIS.md`)
- `CLAUDE.md`, reports, and any `.md` documentation
- Git commit messages
- Content in `.claude/rules/`, `.claude/agents/`, `.claude/skills/`
- Descriptive names in `@todo`, `@note`, `@see`
- Claude's responses to the user

## Allowed exceptions / Excepciones permitidas

The following elements **may** be in English regardless of language setting:
- Variable names, functions, classes, and types in code
- Imports, exports, and language syntax
- Library and dependency names
- User-facing UI texts (if the app is multilingual)
- Technical error messages (stack traces)

## Spanish orthography (mandatory) / Ortografía española (obligatoria)

**Never omit accents (tildes), ñ, or diaeresis when writing in Spanish.**

## Orthography fix tool / Herramienta de corrección

Use `/corregir-ortografia [path]` or `/fix-spelling [path]` to automatically
fix common accent mistakes in markdown files.

## Agent compliance / Aplicación para agentes

Agents in `.claude/agents/` must write their outputs in the configured language.
Reports generated by skills must follow the configured language.
